01

¿Qué es Wachuflow IT?

Wachuflow IT es un proyecto varios servidores, en pleno desarrollo donde se trabaja con una web en una red interna y se necesita cumplir ciertas condiciones para que le aprueben el proyecto.

Su objetivo es diseñar una red organizada en distintas zonas, con servicios especializados y medidas de seguridad que permitan separar el acceso externo, la administración remota y la red interna corporativa.

Condiciones

  • Tiene que ser intermodular
  • No puede o debe contener tareas de clase
  • Tiene que tener red segmentada
  • Uso de alguna herramienta no vista en clase
  • Tiene que utilizar alguna cloud
02

Necesidad que resuelve

Seguridad

Evita exponer la red interna directamente a Internet mediante una arquitectura segmentada con firewall y DMZ.

Organización

Asigna a cada servidor una función concreta: gateway, apache-web, DHCP, Active Directory y BBDD.

Administración

Se organiza de tal manera que técnicos y administradores operan desde la LAN. El administrador de la infraestructura se le permite la conexión remota por VPN para no comprometer la infraestructura.

04

Componentes principales

01

Servidor principal

Basado en Debian 13, actúa como router, firewall y punto central de la infraestructura. Gestiona las interfaces WAN, LAN y DMZ.

  • Firewall con nftables
  • VPN segura con WireGuard
  • Preparado para Telegraf, CasaOS y Dockge
02

Servidor web en DMZ

Servidor Debian 13 ubicado en la red DMZ con Apache en Docker, accesible desde VPN y LAN mediante reglas DNAT del firewall.

  • IP estática: 192.168.100.10
  • Publicación de la web corporativa IT
  • Separación respecto a la LAN para mayor seguridad
03

Servidor DHCP

Servicio encargado de asignar direcciones IP dinámicas a los equipos de la LAN en el rango 192.168.200.100 - 192.168.200.200.

  • Servidor Debian 13
  • IP fija: 192.168.200.10
  • Automatiza la gestión de clientes de red
04

Active Directory

Implementado en Windows Server 2022 como controlador de dominio para gestionar usuarios, grupos y políticas.

  • Dominio: wachuflow.local
  • IP fija: 192.168.200.20
  • Grupos principales: Administradores y Técnicos
05

Base de Datos

Implementado en AWS - RDS con motor de MySQL y conexión vía API con el servidor Web.

  • Tablas: usuarios
  • Capacidad: 20 GB
  • Backups: diarios y gestionados por AWS

06

Seguridad y políticas de comunicación

Reglas clave del firewall

  • Internet → DMZ: permitido solo RDS - AWS
  • Internet → LAN: bloqueado
  • LAN → Internet: permitido
  • LAN → DMZ: permitido
  • DMZ → LAN: bloqueado
  • VPN → LAN y DMZ: permitido para administración

Ventajas de este enfoque

  • Aislamiento del servidor público respecto a la red interna
  • Control estricto del tráfico entre zonas
  • Acceso remoto seguro para personal autorizado
  • Mayor facilidad para escalar la infraestructura
  • Diseño alineado con entornos empresariales reales
07

Flujo de funcionamiento

1

Conexión VPN

Una petición a través de VPN llega por la interfaz del servidor principal.

2

Inspección del firewall

nftables analiza el tráfico y solo redirige el permitido hacia la DMZ.

3

Publicación del servicio web

El servidor Apache en Docker atiende las peticiones a la web corporativa.

4

Conexión vía API

El contenedor de Apache está coenctado vía API a la BBDD.

5

Creación de usuarios

Si se crea un usuario web, éste se queda guardado en la BBDD.

6

Gestión y escalabilidad

La infraestructura está preparada para monitorización y gestión visual con Dockge y CasaOS.
